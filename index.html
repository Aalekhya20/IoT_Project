Introduction of the Project



This project presents a simple yet effective solution for monitoring air quality usingInternet of Things (IoT) technology. The system integrates a low-cost air quality sensor with IoT devices to provide real-time data on air pollutants. The sensor measures key air quality parameters such as particulate matter, carbon monoxide (CO) etc.

The collected data is transmitted wirelessly to a cloud-based platform for analysis and visualization.

The hardware setup consists of a compact and affordable air quality sensor (MQ- 135, MQ-7, MQ-2) connected to an ESP32, equipped with Wi-Fi. The IoT-enabled sensor continuously collects air quality data and sends it to the cloud server. Users can access this information through a user-friendly interface, enabling them to monitor air quality trends, set threshold alerts, and make informed decisions to improve their surroundings.

This project addresses the growing concern of air pollution by providing a cost- effective and accessible solution for individuals, communities, and local authorities to monitor and manage air quality. The simplicity of the system makes it suitable fordeployment in various settings, fostering environmental awareness and encouraging proactive measures to enhance air quality for healthier living environments.

Explanation


This Arduino code is designed for an Internet of Things (IoT) project using an ESP32 microcontroller, WiFi connectivity, and the Blynk platform. The code enables the monitoring of gas sensor data (MQ135, MQ7, MQ2) and sends the readings to the Blynk app for visualization.
The program begins by including necessary libraries for WiFi and Blynk integration.
Users need to replace placeholders with their WiFi credentials and Blynk authentication token. Analog pins are defined for three gas sensors (MQ135, MQ7, MQ2).

In the setup function, the ESP32 connects to the specified WiFi network and initializes communication with the Blynk server using the provided credentials. The loop function continuously reads analog values from the gas sensors, sends this data to corresponding virtual pins on the Blynk app, and prints the values to the Serial Monitor.

Each gas sensor's value is sent to a unique virtual pin (V0 for MQ135, V1 for MQ7, V2 for MQ2). Additionally, the Serial Monitor displays the sensor readings. The Blynk.run() function allows the Blynk library to handle communication with the Blynk server.

A delay of 5000 milliseconds (adjustable based on requirements) is incorporated to control the frequency of sensor readings. Overall, this code facilitates the remote monitoring of gas sensor data through the Blynk app, providing a basis for creating an IoT-enabled environmental monitoring system.
